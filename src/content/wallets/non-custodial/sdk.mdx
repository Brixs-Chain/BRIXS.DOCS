# Quickstart: SDK

Connecting a decentralized application (dApp) to a non-custodial wallet via the Brixs dApp SDK is the standard way to build on our network.

## Complete Implementation Guide

### 1. Installation

Install the React hooks and UI components. We leverage TanStack Query for optimal state management.

```bash
npm install @brixs/dapp-kit @tanstack/react-query
```

### 2. UI Integration

Simply import the `ConnectButton` component. This handles wallet discovery, network switching, and persistent sessions automatically.

```tsx
import { ConnectButton } from '@brixs/dapp-kit';

export function Header() {
  return (
    <header className="flex justify-between p-4 bg-black text-white">
      <h1>My Web3 App</h1>
      <ConnectButton />
    </header>
  );
}
```

## Transaction Signing Flow


### Phase 1: Engine Initialization
The first phase requires a comprehensive setup of the execution environment. <Keyword>BrixsVM</Keyword> allocates memory blocks and establishes secure P2P connections via the gossip protocol. This ensures that when <Keyword>Brixs Sequencer</Keyword> begins proposing blocks, there are no I/O bottlenecks. 

```bash
# Initialize the Brixs node
brixs deploy
```

### Phase 2: State Merklization
Raw blockchain data is ingested into the <Keyword>Brixs Trie</Keyword>, our lock-free, highly concurrent sparse Merkle tree built over RocksDB. Hot state is cached in RAM to guarantee microsecond retrieval times. 

```javascript
// Connect to the high-throughput RPC
const provider = new BrixsProvider("http://localhost:8545");
const block = await provider.getLatestBlock();
console.log(`Current Block: ${block.number}`);
```

### Phase 3: Consensus & Finality
Transactions are mapped into a DAG. <Keyword>BrixsVM</Keyword> dispatches non-conflicting transactions to the thread pool for parallel execution. Once processed, <Keyword>Brixs Sequencer</Keyword> pipelines the block proposal, achieving Instant Rollup Finality in under 400ms.
